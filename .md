# PHP E-commerce API

## Overview

before you read this know that this codebase has been updated over time so i will advice u look at the codebase before coming here

This project is a RESTful API built with PHP and MySQL using the PDO extension. It provides backend services for a simple e-commerce platform, handling user authentication, product management, order processing, and address storage.

## Features

- **PHP**: Core server-side logic for handling API requests and business operations.
- **MySQL (via PDO)**: Secure and prepared statement-based database interaction for data persistence.
- **JSON API**: Standardized data interchange format for all client-server communication.
- **Session-Based Authentication**: Manages user login state to secure protected endpoints.
- **Transactional Integrity**: Ensures data consistency for critical operations like payments and order creation using SQL transactions.

## Getting Started

### Installation

1.  Clone the repository to your local machine or server.
    ```bash
    git clone <repository-url>
    ```
2.  Set up a local web server environment (e.g., XAMPP, MAMP, or a Docker container) with PHP and MySQL.
3.  Create a MySQL database with the name specified in the connection file.
4.  Set up the required database tables. You will need tables for `users`, `products`, `orders`, `order_items`, `addresses`, and `wallet_transactions` based on the queries in the source code.
5.  Update your database credentials in `conn.php` and `auth/conn.php`.

### Environment Variables

This project uses hardcoded variables in `conn.php`. For production environments, it is strongly recommended to move these to a `.env` file or environment variables.

- `$host`: The database host.
  - Example: `localhost`
- `$db`: The name of the database.
  - Example: `preciou3_jek`
- `$user`: The database user.
  - Example: `preciou3_admin`
- `$pass`: The database user's password.
  - Example: `Apcodesphere001`
- `$charset`: The character set for the database connection.
  - Example: `utf8mb4`

## API Documentation

### Base URL

The API base URL is the root directory where the project is hosted.
`http://your-domain.com/`

### Endpoints

#### POST /auth/register.php

Registers a new user in the system.

**Request**:

```json
{
  "name": "John Doe",
  "email": "john.doe@example.com",
  "password": "SecurePassword123",
  "type": "customer"
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Registration successful",
  "user": {
    "id": "123",
    "name": "John Doe",
    "email": "john.doe@example.com",
    "type": "customer"
  }
}
```

**Errors**:

- `400 Bad Request`: `All fields are required`
- `409 Conflict`: `Email already exists`
- `500 Internal Server Error`: `DB error: [message]`

---

#### POST /auth/login.php

Authenticates a user and creates a session.

**Request**:

```json
{
  "email": "john.doe@example.com",
  "password": "SecurePassword123"
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Login successful",
  "user": {
    "id": 1,
    "name": "John Doe",
    "email": "john.doe@example.com",
    "role": "customer",
    "oauth_provider": null,
    "oauth_id": null,
    "created_at": "2023-10-27 10:00:00",
    "session_id": "r9qg8gsvk2t3j5a6pbfm7h1l0k"
  }
}
```

**Errors**:

- `400 Bad Request`: `Email and password required`
- `401 Unauthorized`: `Invalid credentials`
- `500 Internal Server Error`: `DB error: [message]`

#### POST /addresses/add.php

Adds a new delivery address for the authenticated user. Session authentication is required.

**Request**:

```json
{
  "label": "Home",
  "address": "123 Main Street",
  "city": "Anytown",
  "state": "Anystate",
  "country": "Nigeria",
  "postal_code": "100001"
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Address added successfully",
  "address_id": "42"
}
```

**Errors**:

- `400 Bad Request`: `Address, city, and state are required`
- `401 Unauthorized`: `Not authenticated`
- `500 Internal Server Error`: `DB error: [message]`

#### POST /dashboard/add_product.php

Adds a new product to the system. Session authentication is required.

**Request**:

```json
{
  "name": "Organic Apples",
  "description": "Freshly picked organic apples.",
  "price": 5.99,
  "category_id": 1,
  "image_base64": "data:image/jpeg;base64,/9j/4AAQSkZJRgABAQ..."
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Product added successfully"
}
```

**Errors**:

- `400 Bad Request`: `Name, Price and Category are required`
- `400 Bad Request`: `Invalid image type` or `Invalid base64 image format`
- `401 Unauthorized`: `Unauthorized`
- `405 Method Not Allowed`: `Invalid request method`
- `500 Internal Server Error`: `DB error: [message]` or `Failed to save image`

---

#### POST /orders/create_order.php

Creates a new pending order with specified items.

**Request**:

```json
{
  "user_id": 1,
  "address_id": 42,
  "order_type": "single",
  "items": [
    { "product_id": 101, "quantity": 2 },
    { "product_id": 105, "quantity": 1 }
  ]
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Order placed successfully",
  "order_id": 78,
  "total_amount": 150.75
}
```

**Errors**:

- `400 Bad Request`: `user_id and items are required`
- `404 Not Found`: `Product ID [id] not found`
- `500 Internal Server Error`: `Failed to place order`

---

#### POST /dashboard/modify_order.php

Modifies an existing order. Supports canceling an order, deleting an item, or updating an item's quantity. Session authentication is required.

**Request (Cancel Order)**:

```json
{
  "order_id": 78,
  "action": "cancel"
}
```

**Request (Delete Item)**:

```json
{
  "order_id": 78,
  "action": "delete_item",
  "item_id": 210
}
```

**Request (Update Item)**:

```json
{
  "order_id": 78,
  "action": "update_item",
  "item_id": 210,
  "quantity": 3
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Order canceled successfully"
}
```

**Errors**:

- `400 Bad Request`: `order_id and action are required`
- `400 Bad Request`: `Cannot cancel a paid order`
- `400 Bad Request`: `Insufficient stock for product`
- `400 Bad Request`: `Invalid action`
- `401 Unauthorized`: `Not authenticated`
- `404 Not Found`: `Order not found`

---

#### POST /orders/pay_order.php

Processes payment for an order using the user's wallet balance. Session authentication is required.

**Request**:

```json
{
  "order_id": 78
}
```

**Response**:

```json
{
  "status": "success",
  "message": "Payment successful. Your wallet has been debited and farmers credited.",
  "order_id": 78,
  "amount_paid": 150.75
}
```

**Errors**:

- `400 Bad Request`: `Order ID is required`
- `400 Bad Request`: `Order has already been paid`
- `400 Bad Request`: `Insufficient balance. Please top up your wallet`
- `401 Unauthorized`: `Not authenticated`
- `404 Not Found`: `Order not found`
- `500 Internal Server Error`: `Payment failed: [message]`
